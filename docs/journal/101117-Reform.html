2010-11-17 风格变更

在实际使用 C-- 风格代码时，有一个大家一致觉得很麻烦的地方就是，要加 ref 的地方太多。例如：
ref<X> CombineX ( ref<X> a, ref<X> b );

希望能去掉 ref 直接写成：
X CombineX ( X a, X b );

因为变量定义使用的次数远高于类定义，因此我们准备对之前的代码风格进行改造。

调整思路

一种解决方法是将名字直接给引用，即：
class _X;
typedef ref<_X> X;

这样的话，在如下场合会引起一定的不便，因为要使用类本身的名字：

  • 构造函数
  • 继承
  • 静态成员（函数、变量和类型）

但目前尚未找到更好的解决方法。

风格示例

定义类和实例化：

class Ref(X) { ... };
typedef ref<Ref(X)> X;

X CreateX() { return gc_new<X>(); }
bool CompareX(X lhs, X rhs);

构造函数和析构函数：

class Ref(X)
{
    Ref(X)();
    Ref(X)(int);
    Ref(X)(X rhs);
    ~Ref(X)();
};

继承：

class Ref(X) {...};
class Ref(Y): public Ref(X) {...};

X x = gc_new<Y>();
Y y = ref_cast<Y>(x);

静态成员：

class Ref(X)
{
    static void f();
};

Ref(X)::f();

总体来说，这并不是一种好的代码风格。

结论

暂时先搁置这方面的工作，还是显式地使用 ref 来标识引用。

在实际使用时，可以约定一个 typedef 来指定引用类型（在类名末尾加下划线），例如：

class X { ... };
typedef ref<X> X_;
X_ x = gc_new<X>();

其中，gc_new 可以接受类名 X，也可以接受引用类 X_。但建议不要使用后者。
